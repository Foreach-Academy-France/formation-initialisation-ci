# Évaluation - Formation CI/CD GitHub Actions

## Structure de l'évaluation

| Type | Poids | Description |
|------|-------|-------------|
| Contrôle continu | 50% | TPs notés (5 x 10%) |
| QCM final | 50% | 30 questions - 45 min |

## QCM Final

Voir [qcm.md](./qcm.md) pour les questions.

### Format
- 30 questions à choix multiples
- Durée : 45 minutes
- Une seule réponse correcte par question
- Pas de points négatifs

### Thèmes couverts
- Concepts CI/CD (6 questions)
- Syntaxe YAML et workflows (6 questions)
- Triggers et événements (4 questions)
- Jobs et matrix (4 questions)
- Tests et qualité (4 questions)
- Secrets et variables (3 questions)
- Déploiement (3 questions)

## Grille d'évaluation des TPs

Voir [grille-evaluation.md](./grille-evaluation.md) pour les critères détaillés.

### Critères généraux (pour chaque TP)

| Critère | Points |
|---------|--------|
| Fonctionnalité | 40% |
| Bonnes pratiques | 30% |
| Qualité du code | 20% |
| Documentation | 10% |

## Validation

**Note minimale requise** : 10/20

**Compétence validée** : C30 - Piloter le développement technique d'une solution digitale (BC05)
